# [S1] AI报刊生成功能设计

## 需求概述

为news-browser项目添加AI生成报刊功能，允许用户自定义AI配置和提示词模板，生成HTML格式的报刊并保存到工作目录。

## 功能规格

### 1. AI配置管理

**配置存储**：单独的JSON配置文件（`ai-config.json`），存储在应用配置目录。

**配置字段**：
- `api_endpoint`：AI API完整端点URL（如 `https://api.openai.com/v1/chat/completions`）
- `api_key`：API密钥
- `model_name`：模型名称（如 `gpt-4`、`claude-3-sonnet-20240229`）
- `temperature`：生成温度（0-2，默认0.7）
- `max_tokens`：最大token数（默认4000）

**配置界面**：在现有设置弹窗中添加"AI配置"标签页。

### 2. 提示词模板管理

**默认模板**：
```
关注当<周期>的重要动态,筛选 20-30 条有价值的信息，生成<报告名称>的<周期>报特别详细说明事件内容及值得关注的原因，生成的新闻html格式+浅色模式，放到<工作目录>文件夹中，文件名格式：<文件名格式>
```

**占位符说明**：
- `<周期>`：根据设置中的period填充（日/周/月）
- `<报告名称>`：使用设置中的app_name
- `<工作目录>`：使用设置中的work_dir
- `<文件名格式>`：使用设置中的file_pattern

**自定义模板**：用户可以在设置中自定义提示词模板，如果为空则使用默认模板。

### 3. 生成功能

**触发方式**：主页面头部的"生成报刊"按钮。

**确认对话框**：点击按钮后显示确认对话框，显示将要使用的提示词预览。

**生成流程**：
1. 用户点击"生成报刊"按钮
2. 显示确认对话框，预览提示词
3. 用户确认后，调用Tauri后端命令
4. 后端读取AI配置和提示词模板
5. 后端构建完整提示词，调用AI API
6. 后端接收AI响应，生成HTML文件
7. 后端保存文件到工作目录
8. 前端显示成功消息，自动刷新报告列表

**进度显示**：
- "正在准备生成..."
- "正在调用AI API..."
- "正在生成HTML内容..."
- "正在保存文件..."
- "生成完成！"

**错误处理**：显示具体错误信息（API调用失败、网络错误、文件保存失败等）。

### 4. 技术实现

**前端**：
- 在SettingsModal中添加AI配置标签页
- 在App.tsx头部添加生成按钮
- 添加确认对话框组件
- 添加生成状态显示

**后端**：
- 添加AI配置读写命令
- 添加提示词模板读写命令
- 添加AI生成命令
- 实现AI API调用逻辑
- 实现HTML文件生成和保存

**配置文件**：
- `ai-config.json`：AI配置
- `prompt-template.json`：提示词模板配置

## 界面设计

### 设置弹窗

现有标签页：
- 基本设置（现有）

新增标签页：
- AI配置
- 提示词模板

### 主页面头部

在现有按钮区域添加：
- "生成报刊"按钮（带加载状态）

### 确认对话框

显示：
- 提示词预览
- 取消/确认按钮

## 数据流

1. 用户配置AI参数和提示词模板
2. 用户点击生成按钮
3. 前端显示确认对话框
4. 用户确认后，前端调用Tauri命令
5. 后端读取配置和模板
6. 后端构建提示词并调用AI API
7. 后端生成HTML并保存文件
8. 后端返回结果给前端
9. 前端显示状态并刷新列表

## 错误场景

1. AI配置不完整：提示用户配置AI参数
2. API调用失败：显示API错误信息
3. 网络错误：显示网络连接问题
4. 文件保存失败：显示文件系统错误
5. AI响应格式错误：显示解析错误

## 验证标准

1. 用户能够配置AI参数
2. 用户能够自定义提示词模板
3. 点击生成按钮后显示确认对话框
4. 确认后能够成功调用AI API
5. 生成的HTML文件保存到正确位置
6. 生成完成后报告列表自动刷新
7. 错误情况下显示正确错误信息
